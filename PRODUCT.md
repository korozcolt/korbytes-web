# Product

## Register

brand

## Users

**Decisores técnicos y fundadores en LATAM** que necesitan software operativo real — no prototipos, no MVPs sin estructura, no promesas. Son gerentes de operaciones, CTOs de empresas medianas, dueños de comercios con procesos que ya no caben en hojas de cálculo, directores de instituciones con trazabilidad regulatoria. Están cansados de promesas digitales sin sustancia. Buscan un partner técnico que entienda operación, no solo código.

**Contexto de visita:** Llegan por referencia, LinkedIn, búsqueda orgánica o casos publicados. Entran con escepticismo sano: "otra agencia de software más". La web debe romper ese escepticismo en los primeros 3 segundos con evidencia visual real, no con decoración.

## Product Purpose

KOR Bytes S.A.S. existe para convertir operaciones reales en sistemas digitales que funcionan. No vende "desarrollo de software" como commodity — vende orden operativo: procesos, trazabilidad, roles, estados, pagos, reportes. El ecosistema PASS es la hipótesis estratégica: productos verticales reutilizables sobre un núcleo común, en vez de reconstruir la misma base para cada cliente.

**La web debe comunicar esto sin diapositivas.** El mensaje central: "No hacemos software. Ordenamos operaciones." Cada sección debe ser una pieza de evidencia, no una promesa.

## Brand Personality

- **Técnico** — Precisión sobre volumen. Lenguaje de arquitectura, no de marketing. Las decisiones se explican, no se decoran.
- **Ambicioso** — Desde Sincelejo para LATAM. Sin complejo regional. La ambición está en el alcance de los productos (PASS cubre 11 verticales), no en adjetivos.
- **Profesional** — Confiabilidad estructural. Cada línea de código, cada decisión de arquitectura, cada deploy tiene trazabilidad. No hay "magia".
- **Creativo** — La creatividad está en cómo se resuelven problemas operativos complejos con patrones reutilizables, no en efectos visuales gratuitos.

**Voz:** Directa, específica, sin humo. "Este caso conectó VTEX con ICG mediante middleware con trazabilidad de estados" — no "soluciones innovadoras para el retail omnicanal".

## Anti-references

1. **Templates de ThemeForest** — Cards genéricas con iconos de stock, gradients purple-to-blue, animaciones decorativas sin propósito, layouts de "agencia digital" prefabricados.
2. **Agencia genérica estilo "startup de San Francisco"** — Glassmorphism en todo, ilustraciones vectoriales abstractas, métricas infladas ("+300% ROI"), copy lleno de buzzwords, footer sin identidad real.
3. **AI-generated default aesthetic** — Gradiente de texto en todos los headings, glass cards idénticas, eyebrow en cada sección, orbit/globe decorativo sin significado, paleta monocromática cyan-purple.
4. **Banco/law firm impersonales** — Demasiado serios, sin evidencia visual, puro texto corporativo, sin mostrar producto real.

## Design Principles

1. **La evidencia manda** — Cada sección debe mostrar algo real: un screenshot, un diagrama de arquitectura, un nombre de producto, un resultado concreto. Si no hay evidencia, la sección no existe.

2. **Una idea por superficie** — Cada sección comunica exactamente UNA cosa. El Hero dice quiénes somos. Servicios dice qué hacemos. Proyectos muestra evidencia. PASS muestra el ecosistema. Sin diapositivas de 5 bullets.

3. **Desde Sincelejo, sin complejo** — La identidad regional es un hecho, no un eslogan. Se comunica con presencia real (mapa, fotos, contexto local), no con banderas ni clichés tropicales.

4. **Técnico no es frío** — La precisión técnica convive con calidez humana. Colores profundos pero con un acento cálido. Tipografía técnica pero con ritmo. Copy directo pero respetuoso.

5. **Inolvidable por utilidad, no por efectos** — Lo que el usuario recuerda es el diagrama PASS, el caso NEXUS, la claridad del mensaje. No un efecto parallax o un blur decorativo.

## Accessibility & Inclusion

- **WCAG AA** mínimo en todo el sitio
- **Contraste 4.5:1** para body text, 3:1 para large text
- **prefers-reduced-motion** respetado: toda animación colapsa a estático o instantáneo
- **Keyboard navigation** completa: focus rings visibles, orden de tab lógico
- **Screen reader**: aria-labels en iconos, roles semánticos, skip-link mantenido
- **Dark mode único** (identidad actual preservada), evaluar light mode toggle futuro

## Strategic Context

KOR Bytes compite en un mercado donde las agencias de software prometen "innovación" y entregan proyectos sin arquitectura. La web debe ser la prueba de que KOR Bytes es diferente: muestra productos reales (PASS), casos documentados (NEXUS, Torcoroma), y un lenguaje técnico creíble. La web no vende servicios — califica prospectos: quien entiende lo que ve, es el cliente correcto.
